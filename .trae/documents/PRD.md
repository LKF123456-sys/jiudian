# 锦程酒店运营管理系统 - 产品需求文档 (PRD)

## 1. 产品概述

锦程酒店运营管理系统是一款面向中小型酒店前台与运营人员的 Web 端管理工具，覆盖客房、客户、订单、入住/退房四大核心业务流程。系统通过数字化手段替代纸质登记本，帮助前台快速办理入住退房、实时掌握房态、查询客户信息与历史订单，从而提升运营效率与客户体验。

- **目标用户**：酒店前台、值班经理、运营管理人员
- **核心价值**：用一套界面完成从客房上架到客户入住退房的全流程管理，降低手工记录出错率，提供实时房态看板与营收统计
- **项目定位**：大二课程设计级别的可运行系统，采用 Vue 2 + Element UI + Spring Boot + MyBatis + MySQL 前后端分离架构，前端使用 Options API 编写，代码结构直观、便于理解与扩展

## 2. 核心功能

### 2.1 用户角色

| 角色 | 登录方式 | 核心权限 |
|------|----------|----------|
| 前台接待 | 账号密码登录（admin/123456） | 办理入住/退房、管理订单与客户、查看客房 |
| 运营经理 | 同上 | 全部功能 + 数据看板 + 客房维护 |

> 说明：本系统为单租户内部系统，不开放注册，账号由系统预置。

### 2.2 功能模块

1. **登录页**：账号密码登录、记住账号、登录校验
2. **首页看板**：今日数据统计、房态概览、最近订单、快捷操作入口
3. **客房管理**：客房列表、新增/编辑/删除客房、房态筛选、房型维护
4. **客户管理**：客户列表、新增/编辑/删除客户、按手机号/姓名搜索、VIP 等级标记
5. **入住退房管理**：办理入住、办理退房、在住列表、费用结算
6. **订单管理**：订单列表、订单详情、状态流转（待入住→已入住→已退房 / 已取消）、订单取消、营收统计
7. **个人中心**：修改密码、退出登录

## 3. 核心流程

### 3.1 办理入住流程

前台在「入住退房 - 办理入住」页面选择客户与空闲客房，填写入住时间、预计退房时间与押金后提交。系统校验客房状态为空闲，自动生成订单（状态：已入住），将客房状态置为入住，跳转至在住列表。

### 3.2 办理退房流程

前台在「入住退房 - 在住列表」中对待退房订单点击「办理退房」，确认实际退房时间。系统按「实际退房时间 - 入住时间」按夜计费（不足 1 夜按 1 夜计），自动计算总费用，扣除押金后显示应补/应退金额，确认后订单状态变为已退房，客房状态恢复为空闲。

### 3.3 订单状态流转

订单创建后状态为「待入住」或「已入住」（办理入住时直接为已入住）。待入住订单可被取消（状态变更为已取消，客房恢复空闲）；已入住订单办理退房后状态变更为已退房。已退房与已取消订单不可再变更。

```mermaid
flowchart TD
    "空闲客房" --> "办理入住" --> "订单状态: 已入住"
    "订单状态: 已入住" --> "办理退房" --> "订单状态: 已退房"
    "订单状态: 已入住" --> "客房状态: 入住中"
    "订单状态: 已退房" --> "客房状态: 空闲"
    "待入住订单" --> "取消订单" --> "订单状态: 已取消"
    "订单状态: 已取消" --> "客房状态: 空闲"
```

    "入住中" -->|"办理退房"| "空闲"

## 4. 用户界面设计

### 4.1 设计风格

- **主色调**：深空蓝 `#1e3a5f`（沉稳商务）+ 金色点缀 `#d4af37`（高端酒店质感）+ 米白底色 `#faf8f3`
- **辅色**：状态色 - 空闲绿 `#52c41a`、入住中橙 `#fa8c16`、维修中灰 `#8c8c8c`、待入住蓝 `#1890ff`、已退房紫 `#722ed1`
- **按钮风格**：主按钮深蓝实心带轻微阴影、次按钮描边、危险按钮红色，圆角 6px
- **字体**：标题用 `思源黑体 / Noto Sans SC` 中黑，正文用 `思源黑体` 常规，数字用 `DIN Alternate / 等宽数字`，标题 18-24px、正文 14px
- **布局**：左侧固定侧边栏（200px）+ 顶部导航栏（64px）+ 主内容区卡片式布局
- **图标**：Element Plus 图标为主，状态用圆点色块
- **氛围**：商务克制、留白充足、卡片轻微阴影、数据看板有数字滚动动画

### 4.3 响应式

- 桌面优先（最小宽度 1280px 体验最佳）
- 1024px 以上自适应，侧边栏可折叠
- 不重点适配移动端（运营系统主要在 PC 使用）

### 4.4 3D 场景

不适用。
