# 锦程酒店运营管理系统

> 基于 Spring Boot + Vue 3 的现代化酒店管理系统（PMS），支持入住登记、退房结算、客房管理、客户关系、财务管理等全流程业务。

## 项目简介

锦程酒店运营管理系统是一个面向中小型酒店的综合性管理平台，覆盖了酒店前台运营的核心业务流程，包括入住登记、退房结算、客房状态管理、客户档案、消费挂账、发票管理、清扫维修派工等功能。系统采用前后端分离架构，界面美观、操作便捷，适合作为酒店日常运营工具或计算机相关专业的毕业设计项目。

## 功能模块

### 🏠 首页仪表盘
- 今日统计：入住数、退房数、在住数、今日营收
- 客房入住率实时展示
- 近7日营收趋势图（ECharts）
- 房态概览（空闲/在住/维修/脏房统计）
- 今日待办提醒（待清扫/待维修/待结账）

### 🛏️ 入住退房办理
- **预约入住**：选择预约订单办理入住
- **散客入住**：快速添加客户并办理入住
- **客户快捷添加**：入住时直接录入新客户信息
- **押金收取**：支持微信/支付宝/现金/银行卡
- **退房结算**：自动计算房费、消费明细、应收金额
- **续住办理**：一键续住，自动关联原订单
- **换房操作**：支持换房并记录差价/原因
- **时间校验**：入住时间不能早于当前时间前15分钟

### 🏨 客房管理
- **房态看板**：楼层视图，颜色区分房态
  - 🟢 空闲（idle）
  - 🔴 在住（occupied）
  - 🟡 脏房（dirty）
  - 🔵 清扫中（cleaning）
  - 🟣 维修中（maintenance）
  - ⚪ 已预约（reserved）
- **客房列表**：支持按房号、房型、状态、楼层筛选
- **客房CRUD**：新增、编辑、删除客房
- **房型管理**：配置房型名称、床型、价格、容纳人数、设施
- **房间操作**：设置维修/取消维修、设置脏房、分配清扫

### 📋 订单管理
- **订单列表**：按状态筛选（待入住/在住/已退房/已取消）
- **订单详情**：客户信息、入住时间、费用明细、支付记录、操作历史
- **消费入账**：迷你吧、洗衣、餐饮、赔偿等消费录入
- **订单查询**：按订单号、客户姓名/手机号、日期范围搜索
- **订单取消**：支持取消未入住预约订单

### 👥 客户管理
- **客户档案**：姓名、手机号、身份证号、性别、生日
- **VIP等级**：普通/银卡/金卡/钻石卡，不同等级对应不同折扣
- **客户标签**：支持自定义标签（常客、需水果、安静房间等）
- **黑名单**：标记问题客户，入住时自动提醒
- **入住历史**：查看客户历次入住记录
- **累计消费/入住次数**：自动统计

### 💰 财务管理
- **收银管理**：押金收取、房费结算、退款、押金退还
- **多种支付方式**：微信、支付宝、现金、银行卡
- **发票管理**：增值税专用发票/普通发票，支持红冲和作废
- **营收统计**：按时间段查询营收数据
  - 营收总额、订单数、平均客单价
  - 按支付方式统计
  - 按房型统计
  - 按日趋势图
- **支付流水**：所有支付/退款记录查询

### 🧹 客房清扫
- **清扫任务列表**：按状态筛选（待分配/已分配/清扫中/待查房/已验收）
- **任务分配**：将清扫任务分配给保洁员
- **任务状态流转**：开始清扫 → 清扫完成 → 查房验收
- **优先级管理**：低/普通/高/紧急
- **退房自动生成**：退房后自动创建脏房清扫任务

### 🔧 维修管理
- **维修工单**：按状态筛选（待派工/处理中/已完成/已验收）
- **工单分类**：设施/电器/水暖/家具/其他
- **派工处理**：分配维修人员，登记维修方案和费用
- **验收确认**：维修完成后验收
- **报修来源**：前台/保洁/工程人员上报

### 📦 收费项目配置
- **分类管理**：迷你吧、洗衣、餐饮、赔偿、其他
- **价格配置**：项目名称、单价、单位、排序
- **启用/停用**：灵活控制可用项目

### 👤 系统管理
- **用户管理**：新增、编辑、启用/禁用用户
- **角色权限**：
  - `admin` 系统管理员：全部权限
  - `manager` 运营经理：全部权限
  - `receptionist` 前台：入住/退房/订单/客户
  - `finance` 财务：收银/发票/报表
  - `housekeeping` 保洁：清扫任务
  - `engineering` 工程：维修工单
- **操作日志**：AOP自动记录所有关键操作，支持查询
- **个人中心**：修改密码、查看个人信息

## 快速开始

### 环境要求

- JDK 17+
- Maven 3.6+
- Node.js 16+
- MySQL 8.0+

### 数据库初始化

1. 创建数据库：
```sql
CREATE DATABASE jchotel DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

2. 执行建表脚本：
```bash
mysql -u root -p jchotel < jchotel-backend/src/main/resources/sql/hotel.sql
```

3. （可选）导入完整测试数据：
```bash
mysql -u root -p jchotel < jchotel-backend/src/main/resources/sql/full_test_data.sql
```

> 测试数据包含：25个系统用户、40间客房、30位客户、50个订单（含各状态）、完整的支付/消费/清扫/维修/发票数据。

### 后端启动

```bash
cd jchotel-backend
# 编译
mvn compile
# 运行
mvn spring-boot:run
```

后端默认运行在 `http://localhost:8080`

配置文件：`src/main/resources/application.yml`，可通过环境变量覆盖数据库配置：
- `DB_HOST`（默认localhost）
- `DB_PORT`（默认3306）
- `DB_NAME`（默认jchotel）
- `DB_USERNAME`（默认root）
- `DB_PASSWORD`（默认root）

### 前端启动

```bash
cd jchotel-frontend
# 安装依赖
npm install
# 开发模式运行
npm run dev
# 打包构建
npm run build
```

前端默认运行在 `http://localhost:808x`（端口自动递增）

## 核心业务流程

### 入住流程
```
预约/散客 → 选择/新建客户 → 选择房间 → 录入入住信息 → 收取押金 → 办理成功
```

### 退房流程
```
选择在住订单 → 核对消费明细 → 自动计算房费 → 结算支付 → 退还押金 → 退房完成 → 自动生成清扫任务
```

### 客房状态流转
```
idle(空闲) → reserved(已预约) → occupied(在住) → dirty(脏房) → cleaning(清扫中) → idle(空闲)
                    ↘ maintenance(维修中) ↗
```

### 清扫任务流转
```
pending(待分配) → assigned(已分配) → cleaning(清扫中) → done(待查房) → inspected(已验收)
```

### 维修工单流转
```
pending(待派工) → processing(处理中) → done(已完成) → verified(已验收)
```

## 房费计算规则

1. **基础房费**：房间单价 × 入住天数
2. **入住天数**：按实际入住晚数计算（退房日12:00前退房，当日不计费）
3. **延迟退房**：超过12:00加收半天房费，超过18:00加收全天
4. **VIP折扣**：
   - 银卡（VIP1）：95折
   - 金卡（VIP2）：9折
   - 钻石卡（VIP3）：85折
5. **换房差价**：换房后按新房型价格计算，多退少补
6. **附加消费**：迷你吧、洗衣、餐饮、赔偿等实时入账

## 安全特性

- 密码BCrypt加密存储，数据库不明文保存密码
- JWT Token认证，支持登录失败锁定（连续5次锁定30分钟）
- 基于角色的接口权限控制（@RequireRole注解）
- 操作日志AOP自动记录，支持审计追溯
- SQL注入防护（MyBatis参数化查询）

## 接口规范

- 统一响应格式：`{code: 200, message: "success", data: ...}`
- 错误码：200成功、400参数错误、401未登录、403无权限、500服务器错误
- RESTful风格URL
- 分页查询统一参数：`pageNum`, `pageSize`

## 常见问题

**Q: 启动后端报错"Port 8080 is already in use"？**
A: 8080端口被占用，可以修改application.yml中的server.port，或关闭占用端口的程序。

**Q: 前端启动后接口请求跨域？**
A: 后端已配置CORS允许跨域，确认后端正常启动在8080端口即可。

**Q: 登录提示密码错误？**
A: 确认数据库已正确初始化。如果使用hotel.sql初始化，初始密码是MD5(123456)，首次登录会自动升级为BCrypt；如果使用full_test_data.sql，密码是123456。

**Q: 如何修改数据库密码？**
A: 修改application.yml中的spring.datasource.password，或设置环境变量DB_PASSWORD。
