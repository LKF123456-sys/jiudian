# 锦程酒店运营管理系统 — 答辩分工文档

## 一、项目概述

**项目名称**：锦程酒店运营管理系统

**项目简介**：一套面向酒店前台人员的运营管理工具，覆盖客房管理、客户管理、入住/退房办理、订单追踪与营收统计等核心业务场景。采用前后端分离架构，前端基于 Vue 3 + Element Plus 构建 SPA 单页应用，后端通过 Spring Boot RESTful API 提供数据服务，MySQL 数据库承担持久化存储。

**团队定位**：刘开放担任组长和技术核心，负责系统中最复杂的业务流程与数据可视化模块，同时对整体架构有全局理解，答辩时作为主要讲解人承担核心展示。李忠泽和史川皓各负责一组独立的资源管理或辅助模块。

**三人分工原则**：每人负责一组相互独立、业务连贯的前端页面模块，同时深度掌握相关的数据库表结构。答辩时各自演示自己负责的模块并讲解对应的表设计与接口联调逻辑。

## 四、数据库设计（核心 5 张表）

**数据库名称**：hotel_db

| 表名 | 中文名 | 关联关系 |
|------|--------|----------|
| t_user | 用户表 | -- |
| t_room_type | 房型表 | -- |
| t_room | 客房表 | type_id → t_room_type.id |
| t_customer | 客户表 | -- |
| t_order | 订单表 | customer_id → t_customer.id / room_id → t_room.id / user_id → t_user.id |

### 4.1 t_user — 用户表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK | 主键自增 |
| username | VARCHAR(50) | 登录用户名 |
| password | VARCHAR(100) | 加密密码 |
| real_name | VARCHAR(50) | 真实姓名 |
| role | VARCHAR(20) | 角色（admin/operator） |
| create_time | DATETIME | 创建时间 |

### 4.2 t_room_type — 房型表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK | 主键自增 |
| name | VARCHAR(50) | 房型名称（如"豪华大床房"） |
| bed_type | VARCHAR(20) | 床型（大床/双床） |
| capacity | INT | 可住人数 |
| create_time | DATETIME | 创建时间 |

### 4.3 t_room — 客房表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK | 主键自增 |
| room_no | VARCHAR(20) | 房间号（如"301"） |
| type_id | BIGINT FK | 关联 t_room_type.id |
| floor | INT | 楼层 |
| price | DECIMAL(10,2) | 每晚价格 |
| status | VARCHAR(20) | idle（空闲）/ occupied（入住中）/ maintenance（维修中） |
| remark | VARCHAR(255) | 备注 |
| create_time | DATETIME | 创建时间 |

### 4.4 t_customer — 客户表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK | 主键自增 |
| name | VARCHAR(50) | 客户姓名 |
| phone | VARCHAR(20) | 手机号 |
| id_card | VARCHAR(18) | 身份证号 |
| gender | CHAR(1) | M（男）/ F（女） |
| vip_level | INT | 0普通 / 1银卡 / 2金卡 / 3钻石 |
| remark | VARCHAR(255) | 备注 |
| check_in_count | INT | 入住次数 |
| create_time | DATETIME | 创建时间 |

### 4.5 t_order — 订单表（核心）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK | 主键自增 |
| order_no | VARCHAR(32) | 系统自动生成订单号（如"JC20260702xxxx"） |
| customer_id | BIGINT FK | 关联 t_customer.id |
| room_id | BIGINT FK | 关联 t_room.id |
| user_id | BIGINT FK | 关联 t_user.id（操作员） |
| check_in_time | DATETIME | 入住时间 |
| expected_check_out_time | DATETIME | 预计退房时间 |
| actual_check_out_time | DATETIME | 实际退房时间 |
| deposit | DECIMAL(10,2) | 押金 |
| total_amount | DECIMAL(10,2) | 总消费金额 |
| status | VARCHAR(20) | pending（待入住）/ checkedIn（已入住）/ checkedOut（已退房）/ cancelled（已取消） |
| remark | VARCHAR(255) | 备注 |
| create_time | DATETIME | 创建时间 |

**ER 关系总结**：

```
t_room_type (1) ────→ (N) t_room
t_user      (1) ────→ (N) t_order
t_customer  (1) ────→ (N) t_order
t_room      (1) ────→ (N) t_order
```

> **答辩时说**："系统共 5 张表，客房关联房型，订单关联客户、客房和操作员三张表，构成完整的关系模型。"

## 七、关键技术点（三人共用，答辩时可讲）

### 7.1 Axios 拦截器（api/request.js）

```javascript
// 请求拦截器：自动携带 Token
service.interceptors.request.use(config => {
  const token = localStorage.getItem('token')
  if (token) config.headers['Authorization'] = token
  return config
})

// 响应拦截器：自动解包 Result + 统一错误提示
service.interceptors.response.use(
  response => {
    const res = response.data       // Result { code, message, data }
    if (res.code !== 0) {
      ElMessage.error(res.message)
      return Promise.reject(res.message)
    }
    return res.data                 // 解包后直接返回 data 层
  },
  error => {
    ElMessage.error(error.message)
    return Promise.reject(error)
  }
)
```

> **答辩时说**："响应拦截器做了两件事：一是自动解包后端统一返回的 Result 对象，二是统一处理错误提示。所以页面里 .then(data => ...) 拿到的直接就是 data 层的数据。"

### 7.2 路由守卫（router/index.js）

```javascript
router.beforeEach((to, from, next) => {
  const authStore = useAuthStore()
  if (to.path === '/login') {
    authStore.token ? next('/dashboard') : next()   // 已登录则跳首页
  } else {
    authStore.token ? next() : next('/login')         // 未登录则跳登录页
  }
})
```

### 7.3 Pinia 状态管理（store/auth.js）

```javascript
export const useAuthStore = defineStore('auth', {
  state: () => ({
    token: localStorage.getItem('token') || '',
    user: JSON.parse(localStorage.getItem('user') || '{}')
  }),
  actions: {
    setAuth(token, user) {
      this.token = token
      this.user = user
      localStorage.setItem('token', token)
      localStorage.setItem('user', JSON.stringify(user))
    },
    clearAuth() {
      this.token = ''
      this.user = {}
      localStorage.removeItem('token')
      localStorage.removeItem('user')
    }
  }
})
```

### 7.4 Vite 代理配置（vite.config.js）

```javascript
server: {
  port: 8081,
  proxy: {
    '/api': {
      target: 'http://localhost:8080',
      changeOrigin: true
    }
  }
}
```

> **答辩时说**："开发时前端跑在 8081 端口，后端跑在 8080 端口，Vite 的代理功能把 /api 开头的请求自动转发到后端，解决跨域问题。"
